封装, this,构造器,包

2017-06-08

学习封装, this,构造器,包的个人笔记

面向对象的三大特性: 封装, 继承, 多态

一,封装性

不封装的危害

外界使用对象的属性时候, 可以任意的赋值. 比如人的年龄给了一个负数.

什么是封装

把该隐藏的隐藏起来, 把该暴露的暴露出来.

初识两种访问控制符, public 和 private

public: 公共的, 用于修饰属性,方法,类, 在任何地方都可以访问

private: 私有的, 用于修饰属性,方法, 只能在本类内访问

封装的分类

  1. 对属性的封装
  2. 对方法的封装

属性封装的步骤:

  1. 对属性私有化private
  2. 提供公共public的set, get方法访问, 也叫做设置器和访问器

set方法:

public void setAge(int age){
		this.age = age;
}

set方法的方法名,命名格式set打头, 后面跟上属性名.

set方法的参数, 参数的类型和属性的类型要一致,

set方法不需要返回值.

this作用, 区分实例变量和形参

get方法:

public int getAge(){
	return age;
}

get方法的方法名, 命名要以get打头, 不需要参数,但是需要返回值.

get方法的返回值类型必须要和属性的类型一致.

对方法的封装

​ 一个类中可以定义很多方法, 但通常并不是所有方法都需要对外界暴露, 只需要暴露一部分方法即可, 那就可以对无需暴露的方法, 使用private修饰成私有的. 外界无法访问.只能在类的内部使用

二, this关键字

this关键字

使用在本类的方法中, 代表调用该方法的对象本身

this的作用

可用于在本类中, 调用本类方法, 属性,和构造器

三,构造器(构造方法)

构造器的作用

  1. 创建对象
  2. 为对象进行初始化

构造器的格式

如： 访问控制符 类名(参数列表){

​

​ }

构造器的注意事项

  1. 构造器名称必须和类名保持一致
  2. 构造器只能调用一次, 并且是在创建对象时.
  3. 构造器没有返回值类型, 因此也不需要return语句
  4. 构造器之间可以构成重载
  5. 如果没有提供构造器, 系统默认提供一个默认无参构造器, 一旦自己定义了构造器, 那么系统无参构造器将不再提供

四:多个构造器重载使用

  1. 定义多个构造器满足不同的需求

  2. 使用this嵌套调用, 减少冗余代码.

  3. 在一个类中,至少有一个构造器不使用this调用, 避免递归调用.

  4. this在构造器中的调用只能写在第一行

注意:this调用构造器时, 不加点. this(参数列表)

五, 包package

包的作用

  1. 区分重复命名
  2. 控制访问权限
  3. 划分项目的结构层次, 通常将功能相近的类, 划分到同一个包中.

package关键字

  1. 用于确定当前的类所在的位置
  2. com.lanou.demo1 每个包的.是一层级目录
  3. 通常用公司域名倒写.(com.公司域名.项目名.模块名)

import关键字

  1. 用于导入某个包下的类
  2. 导入该包下所有的类和接口,用*
  3. 不同包,相同类名, 又在同一类中使用, 需要使用全限定, 或者其中一个全限定.

如：com.lanou.demo1.Student stu = new com.lanou.demo1.Student();
