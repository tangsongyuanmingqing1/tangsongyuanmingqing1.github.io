static, final, 多态, instanceof, Object类

2017-06-20

学习static, final, 多态, instanceof, Object类的个人笔记

一: static 修饰符

static含义:

静态的, 用于修饰属性,方法, 代码块, 内部类

static修饰的属性(静态变量或者类变量)

​ 使用场景:如果某个属性想被类的全部实例共享, 可以把这个属性修饰成是static
A,静态变量随着类的加载而加载, 随着类的消失而消失
B,静态变量被该类所有的对象所共享
C,一旦某个对象改变了该属性值,其他对象的属性值也会随着改变
D,静态变量优先于对象存在的, 所以即可以通过类名调用. 也可以通过对象来调用

实例变量和类变量的区别

  • 内存中的位置不同
  • 生命周期不同

static 修饰的方法(静态方法或类方法)

  • 静态方法也是随着类的加载而加载
  • 静态方法的存在优先于对象,所以可以通过类名.类方法来调用
  • 静态方法中,不能使用非静态成员, 但可以使用静态成员
  • 非静态方法中, 可以使用静态成员
  • 静态方法中也不可以使用this 和 super
  • 非静态方法中,可以嵌套调用静态方法

二, final 修饰符

含义及作用:

最终的, 可以用于修饰变量(成员变量, 局部变量),方法, 类

使用场景

  • final修饰的类不能被继承
  • final 修饰的方法不能被重写
  • final 修饰的变量,一旦被赋值, 值不能改变

全局静态常量

使用public static final修饰的属性为全局静态常量

三, 多态

多态的含义

一类事物的多种表现形态., 比如人这个事物, 有两种形态,男人, 女人

Java中多态的体现形式

  1. 方法的重载和重写
    方法的功能相同, 但是运行的结果的不同
  2. 对象的多态性
    • 父类的引用,指向子类的对象
    • Java程序的运行分为编译时和运行时, 在多态的情况下:
    • 编译时:看等号左边, 看的是父类引用
    • 运行时:看等号右边, 实际运行的是子类重写的父类的方法.
    • 如果, 父类型引用, 调用子类独有方法, 编译时会出错.

多态的前提

  1. 要有继承关系
  2. 要有方法的重写(运行时动态绑定)

引用类型的强制转换

  1. 向上转型, 子类类型转父类类型, 系统自动完成转换
  2. 向下转型 父类类型转子类类型, 需要强制转换

多态的作用

  1. 多态数组, 多态数组中不仅可以存储本类类型的对象, 还可以存储子类类型的对象

    比如:Person[] persons = new Person[5];
    里面可以存Person对象,也可以存Person子类对象.

  2. 多态参数, 屏蔽子类差异, 写出更加通用的代码

五,java.lang.Object类

  1. Object 类是所有类的基类
  2. 一个类没有继承任何类, 默认extends java.lang.Object
  3. 既然Object类是所有类的父类,那么子类可以继承Object类中所有的方法
  4. 如果对Object类中的方法子类不适用,那么子类可以重写父类方法
  5. Object类中常用方法
    • equals 默认是比较的引用
    • toString 打印对象的输出

六, 代码块

代码块属于是类的成员之一.

非静态代码块

//非静态代码块
{
    
}

每次创建对象时, 都会被执行, 代码块的执行时间优先于构造器.

作用: 创建每个对象时都需要被执行的代码, 可以写到非静态代码块.(为共性代码进行初始化).

代码块可以有多个, 执行顺序按照依次执行.

静态代码块

//静态代码块
static{
    
}
  • 静态代码块,随着类的加载而加载, 只执行一次
  • 静态代码块的执行优先于非静态代码块
  • 静态代码块中不能使用非静态成员
  • 静态代码块可以有多个, 依次向下执行
